# Plan Creator Skill

## Description
Create detailed Plan.md files for complex, multi-step tasks. Plans break down objectives into actionable steps with timelines and success criteria.

## Instructions

### 1. When to Create Plans

Create a Plan.md when:
- Task has 3+ steps
- Task spans multiple days
- Task involves multiple stakeholders
- Task has dependencies
- Task requires resource allocation
- User explicitly requests a plan

### 2. Plan Template

```markdown
---
type: plan
project: [Project Name]
created: 2026-01-23T10:00:00Z
status: active
priority: P1
owner: [Owner Name]
due_date: 2026-01-30
---

# Plan: [Project Name]

## Objective
[Clear, measurable goal]

## Background
[Why this project matters, context from Business_Goals.md]

## Success Criteria
- [ ] [Measurable outcome 1]
- [ ] [Measurable outcome 2]
- [ ] [Measurable outcome 3]

## Steps

### Phase 1: [Phase Name]
**Timeline:** [Start] - [End]

- [ ] Step 1.1: [Action]
  - Owner: [Who]
  - Dependencies: None
  - Output: [Deliverable]

- [ ] Step 1.2: [Action]
  - Owner: [Who]
  - Dependencies: Step 1.1
  - Output: [Deliverable]

### Phase 2: [Phase Name]
**Timeline:** [Start] - [End]

- [ ] Step 2.1: [Action]
  - Owner: [Who]
  - Dependencies: Phase 1 complete
  - Output: [Deliverable]

## Resources Required
- [Resource 1]
- [Resource 2]

## Risks & Mitigations
| Risk | Likelihood | Impact | Mitigation |
|------|------------|--------|------------|
| [Risk 1] | Medium | High | [Action] |

## Communication Plan
- Daily: [Update method]
- Weekly: [Review meeting]
- Blockers: [Escalation path]

## Progress Log

### 2026-01-23
- Plan created
- [Initial actions taken]

---
*Generated by Digital FTE*
```

### 3. Plan Location

Save plans to: `AI_Employee_Valut/Plans/PLAN_[project]_[date].md`

### 4. Plan Lifecycle

1. **Draft** - Initial creation, needs review
2. **Active** - Approved and in progress
3. **On Hold** - Paused for external reasons
4. **Completed** - All success criteria met
5. **Cancelled** - Abandoned with reason documented

### 5. Progress Updates

When working on a plan:
- Check off completed steps
- Add entries to Progress Log
- Update status if phase changes
- Flag blockers immediately

### 6. Integration with Other Skills

- **task-processor**: Creates plans for complex tasks
- **ceo-briefing**: Includes active plans in weekly report
- **audit-logger**: Logs plan creation and updates

## Example Usage

```
Create a plan for launching the new product feature by end of Q1
```
